# コントリビューションガイド

## プロジェクト構成とモジュール構成

このリポジトリは、Vim9script 製のファイルエクスプローラプラグインです。

- `plugin/v9filer.vim`: 起動時の薄いエントリポイント。コマンド、グローバルマッピング、autocmd を定義します。
- `autoload/v9filer.vim`: ファイルのオープン、トグル、表示位置への移動を扱う公開オーケストレーション API。
- `autoload/v9filer/*.vim`: 内部モジュール。
  - `state.vim`: バッファローカルな状態ヘルパー。
  - `fs.vim`: ファイルシステムの一覧取得とファイル操作。
  - `icons.vim`: Nerd Font アイコン選択、ユーザー定義ルール評価、色設定の解決。
  - `render.vim`: パンくずリストとツリーバッファの描画。
  - `actions.vim`: バッファローカルなコマンドとマッピングのアクション。
- `doc/v9filer.txt`: ユーザー向けヘルプと動作仕様。

現時点では、専用の `test/` ディレクトリやアセットディレクトリはありません。

## ビルド、テスト、開発コマンド

ビルド手順は不要です。変更は Vim で直接検証してください。

```sh
vim -Nu NONE -i NONE -n --not-a-term -es -S script.vim
```

有用なスモークテストのパターン:

```vim
set rtp^=/path/to/v9filer.vim
runtime plugin/v9filer.vim
V9Filer /tmp
qa!
```

ファイルの作成、リネーム、削除を伴うテストでは、Vim の `tempname()` を使って一時ディレクトリを作成してください。

## コーディングスタイルと命名規則

すべての実装ファイルで `vim9script` を使用してください。モジュールは小さく保ち、責務を明確にしてください。公開 autoload 関数には `Open()` や `RevealCurrentFile()` のように `UpperCamelCase` を使います。スクリプトローカルなヘルパーは、`ParseArgs()` や `BuildTree()` のように、説明的でアクション指向の名前にしてください。

インデントは 2 スペースを使い、不要なコメントは避け、クラス風の抽象化よりも単純な辞書やリストを優先してください。`plugin/v9filer.vim` は薄く保ち、実装ロジックは `autoload/` に置いてください。

## テストガイドライン

現在のテストはアドホックな Vim スクリプトです。モジュールレベルのヘルパーとユーザーワークフローの両方をカバーしてください。

- コマンドの読み込みとバッファセットアップ
- 埋め込みモードとトグルモード
- ツリー描画、ソート、隠しファイル、サフィックス
- 展開、更新、分割オープン、ヤンク、クローズなどのマッピングやアクション
- 一時ディレクトリ内だけで行うファイルシステム操作
- サイドバーのルート内外にあるファイルに対する `:V9FilerReveal` の動作

`assert_equal()`、`assert_true()`、`v:errors` を使用し、失敗時は `cquit` で終了してください。

## コミットとプルリクエストのガイドライン

このワークスペースには Git 履歴がないため、リポジトリ固有のコミット規約は推測できません。`Add reveal support` や `Fix toggle sidebar close` のように、簡潔な命令形のコミットメッセージを使ってください。

プルリクエストには、短い動作概要、実行したテストコマンド、ユーザーから見える動作が変わる場合は `doc/v9filer.txt` の更新内容を含めてください。UI 描画の変更を文章で説明しにくい場合にのみスクリーンショットを含めてください。
